# 昆明学院教务处学生照片采集系统 — 功能说明书（按角色）

本文档按**角色**说明系统功能、入口、权限范围与典型操作。技术部署参见仓库内 `README` / `.env.example` 等说明。

---

## 一、系统概述

本系统用于**学生（人员）照片采集与管理**，主要能力包括：

- 按**照片管理组**与**备案证件号**控制可操作范围；
- **批量上传**照片，按文件名中的证件号识别人员；
- **上传规范校验**（尺寸、清晰度、背景色等，可在后台配置）；
- **人脸比对**：与历史照片比对相似度，不通过则拒绝或进入**待确认**流程；
- 可选 **Oracle 同步**：将照片二进制写入校外 Oracle 表指定字段（由超级管理员配置）；
- **操作审计**（对部分写请求记录日志）；
- **数据看板**（近 7 天统计，按权限范围过滤）。

系统包含两类界面：

| 类型 | 路径前缀 | 说明 |
|------|----------|------|
| **业务前台** | `/login`、`/dashboard`、`/upload/` 等 | 师生业务操作、看板、上传、待确认 |
| **Django 管理后台** | `/admin/` | 用户、管理组、备案号、校验规则、Oracle 配置等 |

## 二、角色体系说明

系统通过 **两类属性** 共同决定权限：

1. **Django 内置**
   - **超级用户**（`is_superuser`）：在 Django 语义下拥有全部权限；本系统中还用于「秘钥配置」页、**Oracle 照片同步配置**等敏感菜单。
   - **职员**（`is_staff`）：可登录 `/admin/`；具体能否增删改某模型还受 Django **模型权限** 影响。

2. **业务角色**（用户模型 `role` 字段）
   - **管理员**（`admin`）
   - **教学秘书**（`secretary`）
   - **普通用户**（`user`）

3. **数据范围**（非超级用户）
   - 用户可绑定 **照片数据管理组**（`photo_scope_group`）。
   - 仅允许操作在该组 **人员编号与组对应** 中已**备案**的证件号（人员编号）相关照片与待确认项。
   - **超级用户**不受此限制，可查看与操作全部数据。

**重要提示（超级用户 + 批量上传）**：业务前台「批量上传」的**接口**要求业务角色为 **管理员**或**教学秘书**。若账号仅为超级用户但业务角色为「普通用户」，可能**能打开上传页却无法成功提交**；建议将需要上传业务的超级用户业务角色设为 **管理员**，或单独使用管理员/秘书账号上传。

## 四、分角色详细说明

### 4.1 超级管理员（超级用户）

**定位**：系统与数据安全最高权限；配置对外同步与加密密钥；可查看与操作**全部**业务数据。

#### 4.1.1 业务前台

- **数据看板**（`/dashboard/`）  
  - 近 7 天上传数、比对参与数、新建基准照数、人脸比对通过率等；统计**不按管理组过滤**（全库范围）。

- **批量上传**（`/upload/`）  
  - 可选择文件夹批量上传；文件名（无扩展名）作为**人员编号**。  
  - 仅处理有权限的数据：超级用户**不受组限制**，可处理任意已存在业务逻辑允许的人员编号（仍受人脸与校验规则约束）。  
  - **再次强调**：上传 **API** 校验业务角色为管理员或教学秘书；若仅为超级用户 + 普通用户角色，请改为 **管理员** 角色或联系运维。

- **待确认**（`/pending/`）  
  - 查看与处理**全部**待确认记录；可接受（入库）或拒绝。  
  - 若启用 Oracle 且配置为「上传时勾选」，接受时同样可配合勾选同步策略（与配置一致）。

- **秘钥配置与备份**（`/superuser/crypto/`）  
  - **仅超级用户**。  
  - 查看当前 Fernet 密钥生效来源（环境变量 / 数据库 / SECRET_KEY 派生）。  
  - 在环境变量未覆盖时，可将 Fernet 密钥**保存到数据库**。  
  - **显示**当前生效密钥、**解密查看** Oracle 连接密码（高危操作）。  
  - **导出 JSON 备份**（可选包含 Oracle 明文密码）。  
  - Django **Admin** 右上角亦有「秘钥配置与备份」链接至此页。

#### 4.1.2 Django 管理后台（`/admin/`）

- **Oracle 照片同步配置**（唯一一条）  
  - 启用/关闭 Oracle 写入；连接参数；目标表、人员键列、照片 BLOB 列；是否要求上传/待确认时手动勾选同步及默认勾选。  
  - 密码在库中**加密存储**；编辑时密码留空表示不修改。

- **超级管理员密钥配置**（`SuperuserCryptoSettings`，若已迁移）  
  - 与前台秘钥页对应的数据存储（单例）。

- **上传照片校验规则**  
  - 文件大小、分辨率、清晰度、背景色规则等。

- **其他模型**  
  - 用户、管理组、人员编号对应、上传批次、人员照片、待确认、质量问题报告、审计日志、系统配置项等；默认超级用户均可管理（可按需通过 Django 权限再收紧）。

- **照片权限与分组**  
  - **照片管理组**、**人员编号与组对应**；可下载 Excel 模板、**Excel 批量导入**（超级用户始终允许导入，不受业务角色限制）。

#### 4.1.3 安全职责建议

- 妥善保管 **SECRET_KEY**、**ORACLE_SYNC_FERNET_KEY**（若使用）、导出的备份 JSON。  
- 变更 Fernet 密钥后，需在 Admin 中**重新保存 Oracle 数据库密码**。  
- 仅为可信人员分配超级用户。

---

### 4.2 管理员（业务角色）

**定位**：本院系/部门侧业务管理；通常具备 **职员**身份以进入 Admin；数据范围默认限制在**本照片管理组备案的证件号**。

#### 4.2.1 业务前台

- **数据看板**：统计仅含**本组备案人员**在近 7 天的数据。  
- **批量上传**：与超级用户类似，但**自动跳过**非本组备案的证件号；返回中可含跳过列表与数量。  
- **待确认**：仅见**本组备案人员**的待确认项；接受/拒绝后数据仍在组内规则下。  
- **上传质量报告**：当次上传中部分照片未过规范时，可跳转查看问题说明与预览图（**仅本人**作为上传者的报告可在报告页打开）。

#### 4.2.2 Oracle 同步（若超级用户已开启）

- 若配置为「需手动勾选」，上传页/待确认页会出现选项；勾选后成功入库才会尝试写入 Oracle。  
- 若为自动同步，无需勾选；以配置为准。

#### 4.2.3 Django 管理后台

- 需 **`is_staff=True`** 且具备相应**模型权限**才能操作各模块。  
- **Excel 批量导入**「照片管理组」「人员编号与组对应」：**超级用户、管理员、教学秘书**可用（仍需具备对应模型的添加/修改权限）。  
- **不可见/不可用**：**Oracle 照片同步配置**（仅超级用户）、**秘钥配置**前台页。

#### 4.2.4 典型工作流

1. 在 Admin 中维护本组 **人员编号与组对应**（或 Excel 导入）。  
2. 使用 **批量上传** 采集照片。  
3. 处理 **待确认** 与 **上传质量报告** 中的问题件。  
4. 通过 **数据看板** 查看近期采集与比对情况。

---

### 4.3 教学秘书（业务角色）

**定位**：与 **管理员** 在**本系统当前实现**中，业务前台权限**一致**（均可批量上传、看板、待确认，且均受**管理组备案范围**约束）。

#### 4.3.1 与管理员的差异（实现层面）

- 二者区分主要用于**组织架构与审计**上的角色标签（如 `department`、显示名称）。  
- **上传 API、上传页入口、Excel 导入权限** 与管理员相同。  
- Admin 中权限仍由 **is_staff** 与 **Django 权限**决定，并非由「教学秘书」四字自动放开所有表。

#### 4.3.2 使用说明

- 若学校制度上秘书与管理员的 Admin 权限应不同，需由超级用户在 Django **用户** 中分别勾选各模型的**权限**，或后续在代码中细化策略。

---

### 4.4 普通用户（业务角色）

**定位**：一般业务人员；**默认不能批量上传**。

#### 4.4.1 业务前台

- **可访问**：登录、首页、**数据看板**、**待确认**（**仅本组备案**范围内）。  
- **不可访问**：**批量上传**页面（会返回无权说明）。  
- **不可调用**：批量上传 API（返回无权限）。  
- **待确认**：可对**有权查看**的条目执行接受/拒绝（与备案范围一致）。

#### 4.4.2 管理后台

- 若 **`is_staff=False`**：无法登录 `/admin/`。  
- 若为职员且被授予权限：可按权限维护部分数据；**不能使用** Oracle 配置、秘钥页、Excel 批量导入（导入仅限超级用户/管理员/教学秘书）。

#### 4.4.3 适用场景

- 仅需**处理待确认**、**查看看板**的岗位。  
- 需上传时，应使用 **管理员** 或 **教学秘书** 账号（或调整角色）。

## 五、跨角色业务流程说明

### 5.1 批量上传与处理链

1. 用户选择文件夹内图片上传；系统按**文件名**解析人员编号。  
2. **备案校验**：非本组备案号跳过（超级用户不跳过）。  
3. **规范校验**：未通过的可生成**质量问题报告**，通过的进入队列。  
4. **人脸与比对**：与历史照片比对；首张照片在允许策略下可作为**基准照**。  
5. 需人工介入时进入 **待确认**；确认接受后正式入库。  
6. 若启用 **Oracle 同步** 且满足配置与勾选条件，在**本系统入库成功后**尝试 `UPDATE` 外部表（失败一般仅记录日志，不撤销本地入库）。

### 5.2 数据看板指标（近 7 天）

- **上传数**、**参与比对张数**、**新建基准照数**、**比对通过率**（相似度阈值与全局配置相关，默认约 0.85）。  
- 非超级用户统计**仅限其管理组备案人员**。

### 5.3 审计

- 对已登录用户的部分 **POST/PUT/DELETE/PATCH** 请求，中间件会写入**审计日志**（路径与方法等；详情以实际实现为准）。
